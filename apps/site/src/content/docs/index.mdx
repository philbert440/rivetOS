## Why RivetOS?

<CardGrid stagger>
  <Card title="One command setup" icon="rocket">
    `rivetos init` walks you through everything вҖ” providers, channels, deployment. Running in minutes, not hours.
  </Card>
  <Card title="Multi-agent mesh" icon="puzzle">
    Run one agent or a fleet. Opus reasons, Grok codes, local models research. They share memory, delegate tasks over mTLS, and collaborate through a mesh network.
  </Card>
  <Card title="Your infrastructure, your rules" icon="setting">
    Docker on your laptop, LXC on Proxmox, Kubernetes in the cloud. Same config, same agents, different providers. No vendor lock-in.
  </Card>
  <Card title="Plugins for everything" icon="add-document">
    Channels (Discord, Telegram, Voice), providers (Anthropic, OpenAI, xAI, Google, Ollama, openai-compat, claude-cli), tools (shell, files, web, MCP) вҖ” all pluggable, all replaceable.
  </Card>
  <Card title="Persistent memory" icon="star">
    Postgres-backed memory with hybrid FTS + vector search, compaction pipelines, and cross-agent recall. Agents learn from experience via skills.
  </Card>
  <Card title="MCP-native" icon="approve-check">
    Expose RivetOS as an MCP server over Streamable HTTP. Memory, skills, web search, and runtime tools available to any MCP client.
  </Card>
</CardGrid>

## Quick Install

```bash
npx rivetos init
```

That's it. The wizard handles the rest.
